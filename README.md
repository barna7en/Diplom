# Социальная сеть (обмен фотографиями) — backend API

Дипломная работа профессии «Python-разработчик с нуля». REST API на Django и Django REST Framework: публикации с изображением, комментарии, лайки, выдача токена через встроенный эндпоинт DRF.

---

## 1. Краткое описание

Пользователи создают посты (текст + обязательное изображение). Список и просмотр поста доступны без авторизации; создание поста, комментариев и лайков — только с токеном. Редактирование и удаление поста — только автор поста. Детальная карточка поста включает вложенные комментарии и поле `likes_count`.

## 5. PostgreSQL через Docker

В проекте **единственная поддерживаемая база** — **PostgreSQL**. Она поднимается **только через Docker** из файла `docker-compose.yml` в корне репозитория.


Остановка (при необходимости):

```bash

```

## 6. Параметры подключения к БД и переменные окружения

Значения по умолчанию совпадают с `docker-compose.yml` и с `social_network/social_network/settings.py`:

| Параметр | По умолчанию | Переменная окружения |
|----------|--------------|----------------------|
| Имя БД | `dj_diplom` | `POSTGRES_DB` |
| Пользователь | `adilet` | `POSTGRES_USER` |
| Пароль | `1` | `POSTGRES_PASSWORD` |
| Хост (с хост-машины) | `localhost` | `POSTGRES_HOST` |
| Порт (на хосте) | `15432` | `POSTGRES_PORT` |

Django читает те же имена переменных (`POSTGRES_*`). Порт **15432** выбран, чтобы не конфликтовать с локальным PostgreSQL на 5432.

**Важно:** другие СУБД и режимы вроде SQLite в этом проекте **не используются** и в инструкции не предусмотрены.

## 7. Миграции

Команды выполняются из каталога `social_network` с активированным виртуальным окружением и **запущенным** PostgreSQL в Docker.

### `python manage.py makemigrations`

- **Когда:** после изменения моделей в коде (новые/изменённые поля, модели).
- **Зачем:** сформировать файлы миграций в приложениях.

```bash
python manage.py makemigrations
```

### `python manage.py migrate`

- **Когда:** первый запуск проекта на чистой БД; после появления новых файлов миграций; после смены ветки с обновлёнными миграциями.
- **Зачем:** применить миграции к базе данных (создать/обновить таблицы).

```bash
python manage.py migrate
```

Просмотр состояния миграций:

```bash
python manage.py showmigrations
```

## 11. Аутентификация по токену (DRF authtoken)

Токен выдаётся стандартным представлением `obtain_auth_token`: **POST** с телом JSON — логин и пароль **обычного пользователя Django** (не суперпользователя обязательно; любой `User` с установленным паролем).

Создать пользователя можно через `createsuperuser`, через админку или `python manage.py shell` и `User.objects.create_user(...)`. Для пользователя при первом успешном запросе токена в таблице токенов создаётся запись автоматически.

Заголовок для защищённых методов:

```http
Authorization: Token <значение_токена>
```

Файл **`token_settings.md`** в репозитории описывает получение токена **ВКонтакте** и **не используется** для авторизации в этом API.

## 12. Сводка эндпоинтов API

Все пути ниже относительно префикса **`/api/`**.

| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| POST | `/api/token/` | без токена | Получить токен (логин + пароль в JSON) |
| GET | `/api/posts/` | без токена | Список постов |
| POST | `/api/posts/` | **токен обязателен** | Создать пост (`multipart`: `text`, `image`) |
| GET | `/api/posts/<id>/` | без токена | Детали поста (комментарии, `likes_count`) |
| PUT, PATCH, DELETE | `/api/posts/<id>/` | токен; править/удалять может **только автор** | Полное/частичное обновление; удаление |
| POST | `/api/posts/<id>/comments/` | **токен обязателен** | Создать комментарий (JSON: `text`) |
| POST | `/api/posts/<id>/likes/` | **токен обязателен** | Поставить лайк (повторный POST не создаёт дубликат) |
| DELETE | `/api/posts/<id>/likes/` | **токен обязателен** | Снять **свой** лайк с поста |

Лайки: **один URL** — **POST** ставит лайк, **DELETE** снимает лайк текущего пользователя.

## 13. Примеры запросов (curl)

Подставьте свой хост/порт при необходимости. В **PowerShell** для вызова настоящего curl используйте `curl.exe` (иначе может сработать псевдоним `Invoke-WebRequest`). В **bash** объедините параметры в одну строку либо продолжайте строку символом `\` в конце строки вместо `^`.

### 13.1 Получить токен

```bash
curl.exe -X POST http://127.0.0.1:8000/api/token/ ^
  -H "Content-Type: application/json" ^
  -d "{\"username\": \"myuser\", \"password\": \"mypassword\"}"
```

Ответ при успехе: `{"token":"…"}`. При неверных данных — `400 Bad Request`.

### 13.2 Создать пост (обязательны текст и файл изображения)

```bash
curl.exe -X POST http://127.0.0.1:8000/api/posts/ ^
  -H "Authorization: Token ВАШ_ТОКЕН" ^
  -F "text=Привет из API" ^
  -F "image=@C:\path\to\photo.jpg"
```

В ответе — объект поста в «детальном» виде (включая `comments`, `likes_count`).

### 13.3 Список постов

```bash
curl.exe http://127.0.0.1:8000/api/posts/
```

### 13.4 Детали поста с `id=1`

```bash
curl.exe http://127.0.0.1:8000/api/posts/1/
```

### 13.5 Создать комментарий к посту `1`

```bash
curl.exe -X POST http://127.0.0.1:8000/api/posts/1/comments/ ^
  -H "Authorization: Token ВАШ_ТОКЕН" ^
  -H "Content-Type: application/json" ^
  -d "{\"text\": \"Отличный кадр!\"}"
```

### 13.6 Поставить лайк

```bash
curl.exe -X POST http://127.0.0.1:8000/api/posts/1/likes/ ^
  -H "Authorization: Token ВАШ_ТОКЕН"
```

Ответ: JSON с полем `created` (`true` если лайк создан впервые, `false` если уже был).

### 13.7 Снять лайк

```bash
curl.exe -X DELETE http://127.0.0.1:8000/api/posts/1/likes/ ^
  -H "Authorization: Token ВАШ_ТОКЕН"
```

Если своего лайка не было — ответ **404**.

## 14. Заметки для разработки (медиа и тесты)

- В режиме **`DEBUG = True`** загруженные файлы раздаются по префиксу **`/media/`** (см. `social_network/social_network/urls.py`); файлы сохраняются в каталог **`social_network/media/`** (`MEDIA_ROOT`).
- Поле **`image`** у поста обязательно при создании; для обновления используйте подходящий метод (`PATCH` может обновить только текст, если так задано в запросе).
- Автотесты: из каталога `social_network` выполните `python manage.py test` (тестовая БД создаётся на том же PostgreSQL).

### Требования к данным (кратко)

Посты состоят из текста и фотографии, сохраняется время создания. Создавать посты могут только авторизованные пользователи; редактировать — только автор. Комментарии — только у авторизованных, в теле комментария — текст и дата. У поста в детальном ответе — список комментариев и число лайков, например:

```json
{
  "id": 1,
  "text": "Новый прекрасный день",
  "image": "/media/posts/example.jpg",
  "created_at": "2024-02-23T02:24:29.338414Z",
  "comments": [
    {
      "author": 2,
      "text": "Круто",
      "created_at": "2024-02-23T05:12:31.054234Z"
    }
  ],
  "likes_count": 20
}
```

### Подсказки (технические)

* **Токен для этого API** — через **POST `/api/token/`** (раздел 11); файл `token_settings.md` относится к **VK** и к данному REST API не относится.
* Для загрузки и отображения фото настройте `MEDIA_URL` / `MEDIA_ROOT` и раздачу медиа в `urls` при разработке ([документация Django](https://docs.djangoproject.com/en/5.0/howto/static-files/#serving-static-files-during-development)).
* Связанные объекты — через `related_name`, количество лайков — через `count()` у связанного менеджера.
* Вложенная сериализация в DRF — [раздел документации](https://ilyachch.gitbook.io/django-rest-framework-russian-documentation/overview/navigaciya-po-api/relations#vlozhennye-otnosheniya).

* Геоданные и геокодинг (например, `geopy`).
